基于WireGuard的国内到国外VPS组网架构

概述与技术栈建议 需求实现： DNS分流 (黑名单模式)：被墙域名通过8.8.8.8走WireGuard隧道解析；国内及所有未匹配的域名，默认通过114.114.114.114直连解析。 流量分流 (黑名单模式)：目标IP在大陆IP列表中的流量直连；目标IP在被墙IP列表中的流量走隧道；所有未匹配的流量默认直连。 安全增强：集成Kill Switch，当隧道意外断开时，自动阻止所有应走隧道的流量，防止真实IP泄露。 自动化与持久化：所有路由和防火墙规则与WireGuard接口生命周期绑定，实现开机自启、自动配置与清理，无需手动干预。 技术栈： WireGuard：VPN隧道（UDP，端口51820）。 Dnsmasq：本地DNS服务器，分流解析。 ipset + iptables + ip rule/route：政策路由分流。 gfwlist2dnsmasq.sh：生成被墙域名黑名单。 cron + bash：自动化更新。 注意事项： 先在测试环境试运行，避免影响生产。 VPS防火墙需允许WireGuard端口：ufw allow 51820/udp。 VPS启用IP转发：echo 'net.ipv4.ip_forward=1' >> /etc/sysctl.conf; sysctl -p。 步骤1: 安装必要工具 VPS (服务器端)： # 2. 更新Dnsmasq域名列表 wget -O /etc/dnsmasq.d/accelerated-domains.china.conf https://github.com/felixonmars/dnsmasq-china-list/raw/master/accelerated-domains.china.conf wget -O /etc/dnsmasq.d/bogus-nxdomain.china.conf https://github.com/felixonmars/dnsmasq-china-list/raw/master/bogus-nxdomain.china.conf # 清空gfwlist ipset，让脚本重新填充 ipset flush gfwlist /usr/local/bin/gfwlist2dnsmasq.sh -s 8.8.8.8 -i gfwlist -o /etc/dnsmasq.d/gfwlist.conf # 3. 更新ipset IP列表 # 使用临时文件避免网络问题导致列表被清空 TMP_FILE=$(mktemp) wget -O "$TMP_FILE" https://github.com/gaoyifan/china-operator-ip/raw/ip-lists/china.txt ipset flush china_ip while read line; do ipset add china_ip $line; done < "$TMP_FILE" rm "$TMP_FILE" ipset add china_ip 114.114.114.114 # 4. 重启Dnsmasq服务以应用新域名列表 systemctl restart dnsmasq echo "Lists updated successfully."
